# Language Specification

## 1. Scope

本仕様は、pmlint が解析対象とする Markdown の構文制約を定義する。

## 2. Heading Recognition

- 対象レベル: H1（`#`）〜 H4（`####`）
- セクション境界: `#`〜`####` で始まる行
- H5（`#####`）以下はセクション本文の一部として扱い、解析対象外とする
- コードブロック内の `#` 行はセクション境界としない

各見出しの本文（content）は、見出し行の次の行から、次の同レベル以上の見出しまたは EOF までとする。
直属の下位見出し（children）はコンテンツから除外する。

## 3. Front Matter

- ファイル先頭の `---` で始まり `---` で終わる YAML ブロックを front matter とする
- ファイル先頭以外に出現した `---` は front matter として扱わない
- front matter が存在しない場合は `null` として扱う（エラーとしない）
- YAML パースに失敗した場合は `parse-error` として報告し、以降の検証をスキップする

## 4. Content Elements

各見出しの本文から以下の要素を検出する。

| 要素 | 検出条件 |
|------|----------|
| テキスト行 | リスト・コードブロック・見出し以外の行 |
| 空行 | 空白文字のみ、または長さ0の行 |
| リスト | `-`、`*`、`+` のいずれかで始まる行のグループ |
| コードブロック | ` ``` ` で囲まれたブロック（コードブロック内はネストしない） |
| 強調 | `*text*`、`**text**`、`_text_`、`__text__` |

## 5. Section Recognition Rules

- 見出しタイトルの正規化: 前後の空白をトリムする
- 大文字小文字の区別: 区別する（`Overview` と `overview` は別のタイトル）
